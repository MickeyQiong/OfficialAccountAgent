# 微信公众号文章发布工作流

一键完成"封面生成 → 排版转换 → 草稿发布"的完整流水线。

## 工作流程

```
┌─────────────────┐      ┌──────────────────┐      ┌─────────────────┐
│  generate_image │ ───→ │  markdown_to_html │ ───→ │   add_draft     │
│   生成封面图片   │      │   Markdown → HTML │      │  发布公众号草稿  │
└─────────────────┘      └──────────────────┘      └─────────────────┘
       ↓                         ↓                         ↓
  输出 mediaId             输出 HTML 字符串           输出 草稿发布结果
```

**重要**：必须严格按照上述顺序执行，每一步依赖前一步的输出。

---

## 分步详解

### 第 1 步：生成封面图

调用 `generate_image` 工具，根据文章主题生成封面图片并自动上传到微信素材库。

**参数**：


| 参数     | 类型   | 必填 | 说明                                       |
| -------- | ------ | ---- | ------------------------------------------ |
| `prompt` | string | 是   | 图像生成提示词，描述期望的封面图内容与风格 |

**最佳实践**：

- 提示词需包含视觉元素描述（构图、色彩、风格）
- 建议添加"简约大气、适合社交媒体分享、无文字覆盖"等约束
- 示例：`"啤酒酿造工艺场景，金黄麦芽与啤酒花，暖色调，简约大气，适合公众号封面"`

**返回值**：`{ content, details: { mediaId } }` — mediaId 用于第3步。

---

### 第 2 步：Markdown 转 HTML

调用 `markdown_to_html` 工具，将 Markdown 文章转换为微信公众号兼容的 HTML 排版。

**参数**：


| 参数       | 类型   | 必填 | 说明                     |
| ---------- | ------ | ---- | ------------------------ |
| `markdown` | string | 是   | 完整的 Markdown 文章内容 |

**支持的 Markdown 语法**：

- `#` `##` `###` — 一/二/三级标题（自动加微信公众号风格颜色 `#8B0000`）
- `**文本**` — 加粗
- `---` — 分割线
- 表格 `| col1 | col2 |` — 自动渲染为带样式的 HTML 表格

**返回值**：`{ content: [{ type: "text", text: htmlString }], details: htmlString }` — HTML 字符串用于第 3 步。

---

### 第 3 步：发布草稿

调用 `add_draft` 工具，将 HTML 文章发布为微信公众号草稿。

**参数**：


| 参数          | 类型   | 必填   | 说明                                        |
| ------------- | ------ | ------ | ------------------------------------------- |
| `title`       | string | **是** | 文章标题                                    |
| `content`     | string | **是** | HTML 格式的文章内容（即第 2 步返回的 HTML） |
| `htmlContent` | string | 否     | 如果 content 已是 HTML 格式，可传入此参数   |
| `author`      | string | 否     | 作者名，默认为`"米奇"`                      |
| mediaId       | any    | 是     | 封面图片上传后返回的media_id                |

**建议传参**：

```
add_draft(
  title: "文章标题",
  htmlContent: "第2步返回的HTML字符串",
  author: "作者名",
  mediaId: "第1步返回的mediaId字段"
)
```

**返回值**：成功时返回草稿发布确认信息及 mediaId，失败时返回详细错误及排查建议。

## 完整示例

假设要发布一篇关于"精酿啤酒入门指南"的文章：

### 输入

文章标题：`精酿啤酒入门指南：从麦芽到杯中`

Markdown 内容：

```markdown
# 精酿啤酒入门指南

## 什么是精酿啤酒

精酿啤酒强调**小批量、独立酿造**，注重原料品质和风味创新。

## 酿造四大原料

| 原料 | 作用 | 常见品种 |
|------|------|----------|
| 麦芽 | 提供糖分和色泽 | 皮尔森麦芽、焦香麦芽 |
| 啤酒花 | 苦味与香气 | 卡斯卡特、西楚 |
| 酵母 | 发酵产生酒精 | 艾尔酵母、拉格酵母 |
| 水 | 溶剂与口感基础 | 软水/硬水 |

## 品鉴技巧

一看色泽，二闻香气，三尝口感。精酿的世界远比工业啤酒丰富。
```

### 执行

按顺序调用三个工具：

```javascript
// 第 1 步：生成封面图
generate_image({
  prompt: "精酿啤酒工艺场景，金黄麦芽与翠绿啤酒花，木质酿造桶，暖色调光线，简约大气风格，适合微信公众号封面，无文字"
})
// → 返回 { details: { mediaId: "..." } }

// 第 2 步：Markdown 转 HTML
markdown_to_html({
  markdown: "# 精酿啤酒入门指南\n\n## 什么是精酿啤酒\n\n精酿啤酒强调**小批量、独立酿造**..."
})
// → 返回 HTML 字符串

// 第 3 步：发布草稿（使用第1步返回的mediaId，使用第 2 步返回的 HTML）
add_draft({
  title: "精酿啤酒入门指南：从麦芽到杯中",
  htmlContent: "第2步返回的HTML字符串",
  author: "啤酒知识小站",
  mediaId: "第1步返回的mediaId字段"
})
// → ✅ 草稿发布成功!
```

## 前置条件

- **`.env` 文件**已配置以下环境变量：

  - `WECHAT_APP_ID` — 微信公众号 AppID
  - `WECHAT_APP_SECRET` — 微信公众号 AppSecret
  - `BASE_URL` — AI 图像生成 API 地址
  - `API_KEY` — AI 图像生成 API 密钥
- **Node.js** 环境已安装依赖（`@earendil-works/pi-coding-agent`、`dotenv`）
- **微信 IP 白名单**已配置：登录公众号后台 → 设置与开发 → 安全中心 → IP 白名单

---

## 常见问题


| 问题                   | 原因                        | 解决方案                                 |
| ---------------------- | --------------------------- | ---------------------------------------- |
| `40164 IP不在白名单`   | 微信API拒绝未授权IP         | 登录公众号后台添加当前服务器IP到白名单   |
| `WECHAT_APP_ID 未设置` | `.env` 文件未配置           | 检查`.env` 文件并确保变量名正确          |
| `图片生成失败`         | BASE_URL/API_KEY 无效       | 检查`.env` 中 API 配置是否正确           |
| `上传封面失败`         | coverImage 格式不对         | 确保传入的是 PNG 格式的 Buffer 数据      |
| `草稿发布返回错误`     | HTML 内容可能包含不兼容标签 | 确保使用`markdown_to_html` 转换后的 HTML |
